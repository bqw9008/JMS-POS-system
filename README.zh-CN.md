# 小型商超 POS 系统（C#）

语言：[English](README.md) | 简体中文

一个面向小商店、便利店、小型商超的 Windows 桌面 POS 系统。目标不是一开始就做得很重，而是先把商品、库存和收银这几条核心流程跑通，再逐步补齐报表、扫码、小票打印等能力。

项目目前还在早期阶段，所以现在更重视结构清晰、功能可持续扩展，而不是一次性堆满功能。

## ✅ 现在能做什么

目前已经完成了一些基础能力：

- WPF 桌面主壳，当前主要模块已迁移为原生 WPF 页面
- 当前主要 WPF 页面支持英文 / 简体中文切换
- 基础本地文件日志，记录启动、初始化和异常
- 首次启动自动初始化本地 SQLite 数据库
- 分类管理，新增/编辑使用弹窗，外层详情只读
- 商品管理，新增/编辑使用弹窗，外层详情只读
- 库存查看与调整
- 原生 WPF 前台收银、购物车、优惠、现金/线上分次收款和找零计算
- 订单保存与库存扣减
- 结算完成后弹出测试用小票预览窗口，便于先验证小票内容
- 销售记录查询、日期筛选、订单明细和汇总统计
- 基础报表：今日销售额、订单数、库存概况、日/周/月销售统计、商品排行
- 可编辑系统设置页，可维护店铺名、数据库路径、小票打印机名，支持数据库路径校验、重置为当前值，并查看语言和日志等路径信息
- 低库存提示
- 基础分层目录结构
- 数据库建表脚本

还在路上：

- 登录和角色
- 扫码枪支持
- 真实小票打印接入
- 报表导出

## 🚀 快速开始

```powershell
git clone https://github.com/bqw9008/JMS-POS-system.git
cd POS-system-cs
dotnet restore
dotnet build
dotnet run
```

程序首次启动时会自动创建本地 SQLite 数据库。建表脚本在 `Data/schema.sql`。

## 🪵 日志

基础应用日志按当前用户写入：

```text
%LOCALAPPDATA%\POS-system-cs\logs\app-YYYYMMDD.log
```

当前日志覆盖程序启动、语言/数据库初始化、未处理异常、WPF 操作失败，以及收银结算结果、库存变更、商品/分类维护等关键业务操作。


## 🧪 测试数据

测试数据默认不会插入。开发时如果需要自动插入测试分类、商品和库存，可以在运行前启用环境变量：

```powershell
$env:POS_SEED_TEST_DATA = "1"
dotnet run
```

正式使用时不要设置这个变量，或者设置为 `0`。

## 🌐 语言

桌面界面支持 English 和简体中文。运行时可以在左侧导航区域通过语言选择框切换。

当前行为：

- 首次启动且没有保存过语言时，会根据系统界面语言自动选择：中文系统使用简体中文，其他系统使用英文。
- 切换语言后会刷新当前 WPF 页面和导航文字。
- 用户手动切换语言后，会保存到当前用户的 AppData 设置中，重启后继续使用该语言。
- 当前主 WPF 页面、设置页显示文本，以及 WPF 主流程里使用到的服务层校验和异常提示都已接入本地化。
- `UI/LegacyWinForms/` 下的归档代码继续作为迁移参考保留，不是当前这轮本地化覆盖范围。

## 🧩 当前模块

可以在左侧导航区域切换 English / 简体中文。

### 前台收银

前台收银是原生 WPF 页面。可以通过编码、条码或唯一匹配的名称加入商品，维护购物车，使用收银快捷键并一次性修改商品数量，输入优惠，点击收款后可分次记录现金或线上支付，自动扣减剩余应收并计算找零；收款窗口中 F7 为现金、F8 为线上，每次收款后会清空实收输入框；结算时保存订单、扣减库存。

当前结算成功后会额外弹出一个“小票预览”窗口，用于测试和确认小票内容。预览中会展示店铺名、订单号、时间、支付方式、商品明细、合计、优惠、实收和找零。这个阶段还只是预览，不会发送到真实打印机。

### 分类管理

可以新增、编辑、查看和删除分类。新增和编辑会打开单独弹窗填写，主页面只读展示选中分类详情。已经关联商品的分类会被保护，避免误删。

### 商品管理

可以维护商品编码、名称、条码、分类、进价、售价、库存预警值和启用状态，并支持安全删除/停用。新增和编辑会打开单独弹窗填写，主页面只读展示选中商品详情。

### 库存管理

可以查看库存、直接设置库存、按数量增减调整库存，也可以快速看到低库存商品。

### 销售记录

可以按日期范围查询订单，查看订单明细，并统计当前查询结果的订单数和金额。

### 统计报表

可以查看今日销售指标、库存概况、日/周/月销售汇总，以及热销和滞销商品排行。

### 系统设置

可以编辑当前店铺名称、数据库路径、小票打印机名称，也可以查看设置文件、语言设置文件、日志目录和运行目录等只读信息。数据库路径会在保存前校验为有效文件路径；点击重置会恢复为当前已保存值。如果修改了数据库路径，保存时会自动初始化目标 SQLite 数据库。

## 🗄️ 数据库

目前使用 SQLite，方便本地运行和开发。默认数据库文件名是 `pos.db`。

核心表：

- `categories`
- `products`
- `stock`
- `users`
- `orders`
- `order_items`
